# Blatt01

## Git Status erklären

```bash
pm-lecture % git status
On branch b03

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   CONTRIBUTING.md
        modified:   homework/b03.md

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        foo.java

no changes added to commit (use "git add" and/or "git commit -a")
```


Der Befehl git status ist ein Befehl von git der den Status einer ausgewählten Codebase-branch anzeigt. 
Es wird angezeigt welche dateien modifiziert, neu hinzugefügt oder gelöscht wurden, seit dem letzten 
commit der Codebase.

In dem Fall des Beispiels ist es die "branch b03", die wir uns anschauen. Die Ausgabe verät uns des 
weiteren das der Nutzer die Dateien `CONTRIBUTING.md` und `homework/b03.md` bearbeitet hat.

Es wird zudem darauf hingwiesen, dass noch keine der Dateien, die modifiziert, hinzugefügt oder gelöscht 
wurden, per `git add <file>` zu dem nächsten commit hinzugefügt wurden. Ebenso weist git darauf hin dass 
`git restore <file>` benutzt werden kann um, eine datei wieder zu ihrer ursprünglichen Form zu führen.

Wenn wir jetzt die Datei foo.java hinzufügen wollen, so dass die gemachten Änderungen in den nächsten commit 
berücksichtigt werden, geben wir einfach `git add <file>` ein für die Datei:

```bash
git add foo.java
```

Mit den obigen Befehl wird nur die Datei `foo.java` in den nächsten commit berücksichtigt. Wenn man dagegen 
alle dateien hinzufügen will, kann man dass durch `git add .` oder `git commit -a` (wodurch der commit 
direkt durchgeführt wird, was fehler bergen kann). 

Um dann zu commiten kann man dann z.B. folgenden Befehl eingeben:

```bash
git commit -m "Commit for foo.java"
```
