Q: Warum sind die von Ihnen formulierten Testfälle relevant?

A: Testen ist wichtig, weil wir durch schlecht getesteten Code und seine Bugs, unmengen an Zeit (und Geld wenn wir als Unternehmer denken) verlieren.

Q: Warum halten Sie die formulierten Testfälle für unterschiedlich?

A: Ich habe in dem Testmethoden-Namen dies bereits versucht darzustellen.
Aber
- t01: t01_addCat_added_one
    - fügt eine Katze hinzu
    - Checkt ob die Anzahl Katzen höher gegangen ist um eine Katze
    - Es sollte sich um eine Katze erhöhen
- t02: t02_addCat_added_two
    - fügt zwei Katzen hinzu
    - Checkt ob die Anzahl Katzen höher gegangen ist um zwei Katzen
    - Es sollte sich um zwei Katzen erhöhen
- t03: t03_addCat_added_more_than_once
    - fügt eine Katze 2 mal hinzu
    - Checkt ob die Katze doppelt oder einmal gewertet wird
    - Korrektes verhalten wäre natürlich einmal
- t04: t04_getCatByName_exists
    - fügt eine Katze hinzu
    - Checkt ob man sie beim Namen bekommen kann
    - Es sollte die instanz der katze zurück gegeben werden.
- t05: t05_getCatByName_exists_not
    - fügt eine Katze hinzu
    - Checkt ob eine Katze die nicht existiert etwas zurückliefert
    - Es sollte nichts returned werden
- t06: t06_getCatByWeight_cat_range_in
    - fügt 2 Katzen hinzu
    - Checkt ob eine Katze zurückgeliefert wird nach gewicht (min und max sind in range)
    - Result sollte nicht Null sein
- t07: t07_getCatByWeight_cat_range_out
    - fügt 2 Katzen hinzu
    - Checkt ob eine Katze zurückgeliefert wird nach gewicht (min und max sind out of range)
    - Result sollte Null sein
- t08: t08getCatByWeight_cat_best_result
    - fügt 3 Katzen hinzu
     - Checkt ob eine Katze zurückgeliefert wird nach gewicht (min und max sind in range)
    - Result sollte die Katze mit am wenigsten gewicht sein
- t09: t09_getCatByName_null_argument
    - fügt eine Katze hinzu
    - sucht nach einer Katze beim namen, aber der Name ist null
    - Ergebnis soll null sein
- t10: t10_getCatByWeight_max_smaller_min
    - fügt eine Katze hinzu
    - sucht nach einer Katze nach gewicht, aber min > max
    - Ergebnis soll null sein

Damit sehen wir dass die Ergebnise Test Methoden alle unterschiedlich sind und andere Test-Zwecke verfolgen.

Hinweis für die beigelieferte .gitignore

Die gitignore bindet explizit per `!gradle/` die wrapper Datei ein, diese wird aber trotzdem durch das 
`*.jar` ignoriert. Der Fix dazu war bei mir `gradle/**` mit einzufügen um die Dateien im Ordner ebenfalls 
explizit einzubinden.
